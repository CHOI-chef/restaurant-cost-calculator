### 2. 발주 예측 계산 오류
**현상**: 발주 금액이 실제보다 약 11배 크게 나옴
**원인**: 자체제조 내부 재료 펼칠 때 ratio 중복 합산
**올바른 계산 예시**:
```
차슈고기(자체제조): 수입미박삼겹 1500g → 완성량(yld) 1000g
레시피에서 차슈고기 250g 사용
→ 수입미박삼겹 필요량 = 250 × (1500/1000) = 375g
→ 월 120개: 375 × 120 = 45,000g/월
→ 주1회(7일): 45,000 × 7/30 = 10,500g 발주량
→ 발주금액: 10,500 × 15원/g = 157,500원
```
**calcOrderDemand() 함수에서 ratio 계산 로직 수정 필요**

- 대분류 분류 + 그룹 헤더 + 최저가 ⭐ 배지

### 자체제조 탭
- 수정 버튼 + 수정 모달
- yield_amount DB 저장/로드
- ⚠️ 수정 저장 버그 있음 (위 참고)

### 주류 탭
- 수정 버튼
- 공병/박스 보증금 시스템
- 목표 원가율 개별 설정
- 실질 원가 = 납품가 - (병보증금 × 입수) - 박스보증금

### 메뉴 레시피 탭
- 저장된 메뉴 목록 카드 + 수정 버튼
- 재료 수량 직접 수정
- 단가 기준 선택 (기본/당일/월평균)
- ⚠️ 조리법 + 사진 업로드 미완성 (아래 참고)

### 대시보드
- 목표 원가율 vs 실제 원가율 비교
- 메뉴/주류 상태 표시 (목표달성/주의/초과)

### 월 매출 탭
- 월별 트렌드 그래프 (3가지 모드)

### 발주 예측 탭
- 총매출 기반 자동 계산 (매출 비중 % → 월 판매량 역산)
- 레시피 기반 재료별 자동 계산
- 업체별 발주서 + 인쇄
- ⚠️ 발주 금액 계산 오류 있음 (위 참고)

### Excel 내보내기
