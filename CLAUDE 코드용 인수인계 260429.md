## 🔴 알려진 버그 / 주의사항

### 89337% 버그 (핵심!)
- **원인**: 자체제조 `yld`(완성량)이 DB에서 0 또는 null → `cc/0 = Infinity`
- **수정 완료**: `calcMenuCostOverride`에서 `(cp.yld && cp.yld > 0) ? cc/cp.yld : cp.perG` 안전처리
- **추가 수정**: `compounds` 로드 시 `Math.max(Number(r.yield_amount)||1, 0.001)`

### type / rtype 혼용
- 신규 추가 재료: `item.type = '식재료'`
- DB에서 불러온 재료: `item.rtype = '식재료'`

### OCR 사용법
1. 단가 관리 탭 > "오늘의 납품 단가 입력" 카드 > **📷 이미지 선택** 클릭
2. Tesseract.js가 한글+영어 인식 (CDN, 첫 실행은 학습 데이터 5MB 로드 필요)
3. 검토 모달에서 자동 매칭된 식재료 확인 → 수정/체크 → "일괄 입력" 클릭
4. 인식률을 높이려면: 선명한 이미지, 정면 촬영, 표 형태 권장
5. 별칭을 등록해두면 매칭 정확도 ↑ (예: "참기름오뚜기" → 오뚜기 참기름)

### OCR 핵심 함수
| 함수 | 역할 |
|---|---|
| `runOcr(event)` | 파일 입력 → Tesseract → parseOcrLines → 검토 모달 오픈 |
| `parseOcrLine(line)` | 한 줄에서 (재료명, 단가, 금액) 추출 휴리스틱 |
| `openOcrReview()` / `closeOcrReview()` | 검토 모달 제어 |
| `bulkAddOcrPrices()` | 체크된 항목들을 `priceHistory`에 일괄 추가 |

## 🔑 핵심 함수 목록
| 함수 | 역할 |
|---|---|
| `loadAllData()` | Supabase에서 전체 데이터 로드 + 전체 렌더 |
| `recalcAllCosts()` | 식재료 단가 변경 시 자체제조+메뉴 원가 재계산 |
| `calcMenuCostOverride(menu, getPU)` | 단가 기준(오늘/월평균) 적용해 메뉴 원가 계산 |
| `getEffectiveItems(items)` | priceMode에 따라 재료별 실효 단가 계산 |
| `renderMenuList()` | 메뉴 레시피 탭의 저장된 메뉴 목록 렌더 |
| `getTodayPU(ingName)` | 오늘 납품가 조회 (없으면 null) |
| `getMonthlyAvgPU(ingName)` | 이번달 평균 납품가 조회 |
| `findIngByNameOrAlias(q)` | 이름/별칭/대분류로 식재료 1건 찾기 (OCR/외부 입력용). 결과: `{item, matched, score}` 또는 null |
| `parseAliases(s)` | 쉼표 구분 문자열 → 배열 |
| `normName(s)` | 공백·기호 제거 + 소문자 (느슨한 매칭용) |
