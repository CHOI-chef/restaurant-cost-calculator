  | `addCookingStep()` / `delCookingStep(i)` / `moveCookingStep(i,dir)` | 단계 추가/삭제/순서이동 |
  | `updateCookingStep(i,field,val)` / `delStepImage(stepIdx,imgIdx)` | 필드 갱신 / 사진 제거 |
  | `renderCookingSteps()` | `#cooking-steps-list` innerHTML 재생성 |
  | `uploadStepImage(stepIdx, fileInput)` | Storage 업로드 → publicUrl을 `images`에 push |
  | `openRecipePrintPreview(idx)` / `closeRecipePrintPreview()` / `printRecipeNow()` | 교육자료 미리보기 모달 + A4 인쇄 |
  | `_buildRecipePrintHTML(menu)` | 메뉴명·재료표·조리단계 카드 HTML 생성 |
- 전역: `cookingSteps` (현재 편집 중 메뉴의 단계 — `rItems`와 동일한 패턴)
- 저장 흐름
  - `saveMenu()` → `obj.cookingSteps` 포함 → `saveMenuToDB()` → `cooking_steps` jsonb 컬럼
  - 컬럼 없을 시 (마이그레이션 미실행) cooking_steps 제외하고 재시도 + console.warn
  - `loadAllData()` → `cookingSteps` 필드로 복원
  - `loadMenuForEdit()` → 깊은 복사로 cookingSteps 복원 후 `renderCookingSteps()`
  - `openMenuEdit()` 모달은 cookingSteps 건드리지 않음 (m.cookingSteps 그대로 유지)
- Storage
  - 버킷: `recipe-images` (public read)
  - 경로: `{user_id}/{menu_name 안전화}/step{N}_{timestamp}_{filename}`
  - RLS: 본인 user_id 폴더에만 INSERT/UPDATE/DELETE, SELECT는 public
  - 메뉴 삭제 시 Storage 사진은 자동 정리 안 됨 (orphan) — 향후 cleanup 필요 시 별도 작업
- 인쇄 격리: `body.printing-recipe` 클래스로 `@media print` 분기. 발주서 인쇄도 `body.printing-order`로 동일 패턴 적용 (Ctrl+P 사고 방지)
- 마이그레이션: `migrations/20260513_cooking_steps_and_storage.sql` — Supabase SQL Editor에서 1회 실행 필요

#### 자체제조 yld 미설정 자동 보정 (2026-05-13 추가)
- **증상**: 차슈 단가 15원 / 발주수량 3,150,000g / 발주금액 47,250,000원 (기대 4,200g·63,000원의 약 750배)
- **원인**: 자체제조 `yld`가 미설정(default `1`)이고 메뉴 레시피의 `recipeQty`가 큰 값(예: 750g of 차슈고기)일 때
  `ratio = recipeQty / yld = 750/1 = 750` 으로 폭발 → 모든 sub 식재료 수량이 N×N배로 부풀려짐
- **수정** (`_expandRecipeItem`)
  - 조건 `c.yld<=1 && recipeQty>5` 시 `ratio`를 `1`로 자동 보정 — sub.qty를 "메뉴 1개당 사용량"으로 해석
  - `console.warn` + 발주서 상단에 노란 경고 배너로 보정된 자체제조 이름 노출 (사용자가 yld 입력하면 자동 정상화)
  - `calcOrderDemand` 반환 배열에 `_cappedCompounds` 부가 속성으로 보정 목록 첨부
- **localStorage 저장 방어** (`renderRevenueBased`/`_saveRevenueInputLS`)
  - 비중 입력값이 새로고침 후 사라지는 환경(private 모드/quota 초과) 대응 — 저장 실패 시 toast로 안내
  - 메모리 상태가 비어있을 때 LS 재로드 시도 (race 방어)
