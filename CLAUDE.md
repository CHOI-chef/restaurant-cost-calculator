# 요식업 원가 계산기 — Claude Code 작업 지침

## 사용자 프로필
- 비개발자입니다. 코딩 지식이 전무한 수준이며, 코드를 읽거나 검증할 능력이 없습니다.
- 외식업 원가관리 SaaS를 AI 도구로 직접 만들고 있습니다.
- 매장 운영 경험 보유. 개발은 처음입니다.

## 의사소통 규칙 (필수)
- 모든 설명·진단·보고는 한국어로 작성합니다.
- 코드를 길게 보여주지 말고, "무엇을/왜" 하는지 일상어로 먼저 설명합니다.
- 전문 용어를 쓸 때는 짧게 풀어서 설명합니다 (예: "커밋 = 변경사항 저장").
- 사용자가 잘못된 전제로 요청하면, 명령어를 만들기 전에 먼저 그 전제를 짚어 줍니다.

## 위험 작업 규칙 (필수)
- 데이터 삭제, force push, 파일 영구 제거 등 위험 작업은 실행 전 반드시 경고합니다.
- 위험 작업에는 되돌리는 방법도 같이 안내합니다.
- git push는 자동으로 실행하지 않고, 사용자가 명시적으로 요청할 때만 실행합니다.
- 커밋은 진행해도 되지만, 커밋 메시지를 보여주고 진행합니다.

## 위험 영역 (특히 주의)
- 자체제조(compounds) 계산 로직 — 가장 버그가 잦은 영역. 수정 시 회귀 테스트 필수.
- 자체제조 중첩 계산 — 메뉴 안에 자체제조가 들어가는 구조. 수율(yield_amount) 처리에 특히 주의.
- 발주 예측 계산 — 자체제조 펼치기 시 비율 중복 합산 주의 (과거 11배 오류 발생).

## 작업 흐름 원칙
1. 사용자가 요청 → 잘못된 전제가 없는지 먼저 확인
2. 작업 전 git status로 현재 상태 확인
3. 변경사항을 한국어로 설명 → 사용자 승인 → 코드 수정
4. 수정 후 어떤 파일이 어떻게 바뀌었는지 한국어로 보고
5. 커밋 (메시지 미리 보여주기) → push는 사용자 확인 후

## 기술 환경
- 프론트엔드: Vanilla JS + HTML/CSS (프레임워크 없음)
- 외부 라이브러리: Supabase JS Client, XLSX (Excel 입출력)
- 작업 도구: PowerShell (Windows)

## 사업 맥락 (참고)
- 출시 1차 타겟: 초보 외식업 사장 (단일 매장)
- 포지셔닝: "정확한 계산 토대 + 운영 가이드" (베테랑용 계산기와 차별화)
- 현재 검증 중인 핵심 기능: 자체제조 중첩 계산, 발주 예측, 조리법 매뉴얼
